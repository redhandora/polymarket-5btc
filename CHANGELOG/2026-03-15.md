# 2026-03-15

## 修复 backtest 无法运行的问题

### 背景
初次运行 `npx tsx src/index.ts --backtest` 遇到多个问题，逐一排查修复。

---

### 1. Node fetch 不走系统代理

**问题**：Node 25 内置 `fetch`（undici）不读取 `HTTPS_PROXY` 环境变量，也不受 `setGlobalDispatcher` 影响，导致连接 Polymarket API 超时。

**修复**：新增 `src/utils/fetch.ts`，封装 undici 的 `fetch` + `ProxyAgent`，读取 `HTTPS_PROXY` 环境变量自动注入代理。所有需要发 HTTP 请求的模块改为从此处导入 `fetch`。

**影响文件**：
- `src/utils/fetch.ts`（新增）
- `src/backtest/historical.ts`
- `src/market-data/polymarket-history.ts`

**运行方式**：
```bash
HTTPS_PROXY=http://127.0.0.1:7897 npx tsx src/index.ts --backtest 1000 2026-03-01 2026-03-07
```

---

### 2. BTC 5-min 市场查询接口错误

**问题**：原代码用 `?tag=btc&closed=true` 查询市场，实际上该参数无效，返回的是随机市场，导致找到 0 个 BTC 市场。

**根因**：Polymarket 的 BTC 5-min 市场（2026-02-12 上线）slug 格式为 `btc-updown-5m-{windowOpenTimestamp}`，需要按 slug 精确查询。

**修复**：`fetchBtcMarketsInRange` 改为枚举时间范围内所有 5-min 时间戳，逐个用 `?slug=btc-updown-5m-{ts}` 查询，分批（每批 20 个并发）执行。同时获取 `clobTokenIds[0]`（"Up" token）供后续价格查询使用。

---

### 3. 价格历史接口错误

**问题**：原代码用 `data-api.polymarket.com/prices/history` + conditionId，返回 404。

**根因**：正确接口是 `clob.polymarket.com/prices-history`，且需要传 token ID（`clobTokenIds[0]`），不是 conditionId；`fidelity` 最小值为 10。

**修复**：`polymarket-history.ts` 改用 CLOB API，`fetchHistoricalPrices` 参数从 conditionId 改为 tokenId，`historical.ts` 中市场数据结构新增 `tokenId` 字段。

---

### 4. 增加 backtest 进度日志

每个处理阶段增加屏幕输出，方便定位卡顿位置：
- 市场获取阶段：每 10 批输出一次进度
